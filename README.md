# Stock Alert Bot

Бесплатный Telegram-бот: алерты при резком падении акции (с попыткой объяснить причину) +
еженедельный новостной дайджест по списку тикеров. Работает на бесплатных тарифах
Telegram Bot API, Finnhub и GitHub Actions — стоимость $0/месяц без AI-объяснений,
несколько центов в месяц с ними.

## Как это работает

- `stock_alert_bot.py` — вся логика: получает цены и новости через Finnhub, шлёт сообщения в Telegram.
- `tickers.json` — список тикеров и порог падения в %. Редактируется без касания кода.
- `.github/workflows/stock_bot.yml` — расписание запуска через GitHub Actions (бесплатный cron).
- `state.json` — создаётся автоматически при первом запуске, хранит «уже алертил сегодня по этому тикеру», чтобы не спамить одним и тем же падением каждые 30 минут.

## Настройка, шаг за шагом

### 1. Создать Telegram-бота
Напиши **@BotFather** в Telegram → `/newbot` → следуй инструкциям → получишь токен вида `123456:ABC-DEF...`. Это `TELEGRAM_BOT_TOKEN`.

### 2. Узнать свой chat_id
Напиши своему новому боту любое сообщение, затем открой в браузере:
`https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates`
В ответе найди `"chat":{"id": ...}` — это число и есть `TELEGRAM_CHAT_ID`.

### 3. Получить ключ Finnhub
Зарегистрируйся на [finnhub.io](https://finnhub.io) (бесплатно) → в дашборде будет API-ключ. Это `FINNHUB_API_KEY`. Бесплатный тариф — 60 запросов в минуту, более чем достаточно для 10-20 тикеров.

### 4. (Опционально) Ключ Anthropic для AI-объяснения причины
Если хочешь, чтобы причину падения формулировал ИИ, а не ты сам читал заголовки — получи ключ на [console.anthropic.com](https://console.anthropic.com) (`ANTHROPIC_API_KEY`). Если не задать эту переменную, бот просто пришлёт заголовки новостей без AI-комментария — всё остальное работает как обычно.

### 5. Залить файлы в репозиторий GitHub
Создай **приватный** репозиторий и добавь туда все файлы из этого комплекта, сохраняя структуру папок — особенно важно, чтобы `stock_bot.yml` лежал именно по пути `.github/workflows/stock_bot.yml`, а не в корне. Через GitHub веб-интерфейс можно нажать «Add file → Create new file» и ввести полный путь в поле имени — папки создадутся автоматически.

### 6. Добавить секреты
В репозитории: **Settings → Secrets and variables → Actions → New repository secret**. Добавь:
- `FINNHUB_API_KEY`
- `TELEGRAM_BOT_TOKEN`
- `TELEGRAM_CHAT_ID`
- `ANTHROPIC_API_KEY` (если используешь AI-объяснения)

### 7. Настроить список тикеров
Отредактируй `tickers.json` — впиши свои 10-20 тикеров и желаемый порог падения (по умолчанию 5%).

### 8. Проверить
Перейди в вкладку **Actions** репозитория → выбери workflow **Stock Alert Bot** → **Run workflow** → выбери режим `check` или `digest` → запусти. Это и есть запуск «по запросу», который ты просил изначально — не нужно ждать расписания.

## Расписание по умолчанию

- Проверка просадок — каждые 30 минут по будням, примерно покрывая торговую сессию NYSE/NASDAQ.
- Дайджест новостей — раз в неделю, по понедельникам утром.

Расписание задаётся в `.github/workflows/stock_bot.yml` обычным cron-синтаксисом, можно менять под себя.

## Известные ограничения (честно)

- GitHub Actions не гарантирует запуск ровно по расписанию — в нагруженные периоды возможна задержка в несколько минут. Для алертов о просадке это не критично.
- Качество «возможной причины» падения зависит от того, есть ли вообще релевантные новости по тикеру за последние 1-2 дня. Если акция падает «за компанию с рынком» без своего триггера, AI честно напишет, что явной причины не видно.
- Бесплатный тариф Finnhub даёт реал-тайм данные по US-биржам; международные тикеры могут требовать платного тарифа.
